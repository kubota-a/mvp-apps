MVP-01：ロール別ToDo管理システム

画面設計やテーブル設計を考える前に、まず下記だけチェック&レビューをお願いします。

---------------------
【クライアントの課題】※課題文より抜粋
誰が何を担当しているのか分からない
管理者が進捗を把握できない
完了したかどうか曖昧
担当変更の履歴が残らない
---------------------
【業務ルール】
課題文で例にあがっていた下記を採用：
ステータス（未着手 / 進行中 / 完了）を持つ

理由：
クライアントの課題「管理者が進捗を把握できない・ToDoが完了したかどうか曖昧」を解決できるため
---------------------
【必須機能】
●一般ユーザー
・ログイン機能
（自分のToDoのCRUD）
・新規作成
・編集
・削除（完全な削除ではなく、未完了ToDo一覧から削除して完了ToDo一覧に表示させる）
・一覧（未完了ToDo一覧・完了ToDo一覧）

●管理者
・ログイン機能
・全ユーザーのToDoを閲覧（未完了ToDo一覧・完了ToDo一覧・件数表示・ステータス絞り込み検索）
---------------------
【完成定義】
ロールごとの必須機能を実装した時点で完成とする
---------------------
【削る項目】
・リマインダー機能
・チーム内でのToDo一覧の共有機能
・ToDoの担当者変更機能
・管理者が「完了ToDo一覧」から差し戻しをかけてToDoを未完了一覧に戻す機能と、それに伴うToDoの管理者（一般ユーザー）への通知機能
・一般ユーザーの「完了ToDoの一覧」を直近1か月に絞って表示する
・締切日設定機能
---------------------
