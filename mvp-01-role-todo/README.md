MVP-01：ロール別ToDo管理システム

画面設計やテーブル設計を考える前に、まず下記だけチェック&レビューをお願いします。

---------------------
【クライアントの課題】※課題文より抜粋
誰が何を担当しているのか分からない
管理者が進捗を把握できない
完了したかどうか曖昧
担当変更の履歴が残らない
---------------------
【業務ルール】
課題文で例にあがっていた下記を採用：
ステータス（未着手 / 進行中 / 完了）を持つ

理由：
クライアントの課題「管理者が進捗を把握できない・ToDoが完了したかどうか曖昧」を解決できるため
---------------------
【必須機能】？必須機能にセキュリティとかは含まない？
●一般ユーザー
・ログイン
（自分のToDoのCRUD）
・新規作成
・編集
・ToDo完了申請
・一覧（未完了ToDo一覧・完了ToDo一覧）

●管理者
・ログイン
・全ユーザーのToDoを閲覧（未完了ToDo一覧・完了ToDo一覧・件数表示・ステータス絞り込み検索）
・全ユーザーのToDo新規作成
・全ユーザーのToDo編集
・全ユーザーのToDo削除
・ToDo完了申請の承認および却下
・一般ユーザーのサインアップ
---------------------
【完成定義】
ロールごとの必須機能を実装した時点で完成とする
---------------------
【削る項目】
●一般ユーザー
・リマインダー機能
・管理者が自分のToDoに変更を加えたときや各種申請の却下が行われた場合などの、通知
・ToDoの担当者変更申請
●管理者
・各種申請が来たときなどの、通知
・ToDo完了以外の各種申請の承認および却下
・部署内でのToDo一覧の共有設定
●全ロール共通
・締切日設定
●セキュリティ
---------------------
